### **Java Basics & Fundamentals**

#### 1. What is Java? Explain its features.
**Tags:** `#Java-Core` `#Basics`  
**Difficulty:** ‚≠ê  
**Answer:** Java is a high-level, object-oriented programming language. Features: Platform independent, Object-oriented, Secure, Robust, Multithreaded, Architecture neutral, Portable, High performance.

---

#### 2. What is the difference between JDK, JRE, and JVM?
**Tags:** `#Java-Core` `#Basics` `#Tricky`  
**Difficulty:** ‚≠ê  
**Answer:**
- **JDK** (Java Development Kit): Complete development kit (JRE + development tools)
- **JRE** (Java Runtime Environment): Runtime environment (JVM + libraries)
- **JVM** (Java Virtual Machine): Executes bytecode

---

#### 3. Explain the Java compilation process.
**Tags:** `#Java-Core` `#Basics`  
**Difficulty:** ‚≠ê‚≠ê  
**Answer:** Source code (.java) ‚Üí Compiler (javac) ‚Üí Bytecode (.class) ‚Üí JVM ‚Üí Machine code ‚Üí Execution

---

#### 4. What is bytecode in Java?
**Tags:** `#Java-Core` `#Basics`  
**Difficulty:** ‚≠ê  
**Answer:** Bytecode is platform-independent intermediate code generated by Java compiler. JVM converts it to machine code.

---

#### 5. Why is Java platform independent?
**Tags:** `#Java-Core` `#Basics`  
**Difficulty:** ‚≠ê  
**Answer:** Java code compiles to bytecode which runs on any platform with JVM. "Write Once, Run Anywhere" (WORA).

---

#### 6. What are the 8 primitive data types in Java?
**Tags:** `#Java-Core` `#Basics`  
**Difficulty:** ‚≠ê  
**Answer:** byte (8-bit), short (16-bit), int (32-bit), long (64-bit), float (32-bit), double (64-bit), char (16-bit), boolean (1-bit)

---

#### 7. What is the difference between int and Integer?
**Tags:** `#Java-Core` `#Basics` `#Tricky`  
**Difficulty:** ‚≠ê‚≠ê  
**Answer:**
- **int**: Primitive type, stores value directly, default 0, cannot be null
- **Integer**: Wrapper class, object, default null, has methods

---

#### 8. Explain autoboxing and unboxing.
**Tags:** `#Java-Core` `#Advanced`  
**Difficulty:** ‚≠ê‚≠ê  
**Answer:**
- **Autoboxing**: Automatic conversion of primitive to wrapper (int ‚Üí Integer)
- **Unboxing**: Automatic conversion of wrapper to primitive (Integer ‚Üí int)

---

#### 9. What is type casting? Explain widening and narrowing.
**Tags:** `#Java-Core` `#Basics`  
**Difficulty:** ‚≠ê  
**Answer:**
- **Widening (Implicit)**: Small to large (byte ‚Üí int), automatic, no data loss
- **Narrowing (Explicit)**: Large to small (int ‚Üí byte), manual, may lose data

---

#### 10. What is the difference between == and .equals()?
**Tags:** `#Java-Core` `#Basics` `#Tricky`  
**Difficulty:** ‚≠ê‚≠ê  
**Answer:**
- **==**: Compares memory addresses (reference)
- **.equals()**: Compares content/values

---

#### 11. Why is String immutable in Java?
**Tags:** `#Java-Core` `#Advanced` `#Tricky`  
**Difficulty:** ‚≠ê‚≠ê  
**Answer:** Security, Thread safety, String pool optimization, Hashcode caching, Performance

---

#### 12. What is String pool?
**Tags:** `#Java-Core` `#Advanced`  
**Difficulty:** ‚≠ê‚≠ê  
**Answer:** Special memory area in heap where String literals are stored. Reuses strings to save memory.

---

#### 13. Difference between String, StringBuilder, and StringBuffer?
**Tags:** `#Java-Core` `#Advanced` `#Tricky`  
**Difficulty:** ‚≠ê‚≠ê  
**Answer:**
- **String**: Immutable, thread-safe, slow for concatenation
- **StringBuilder**: Mutable, not thread-safe, fast
- **StringBuffer**: Mutable, thread-safe, slower than StringBuilder

---

#### 14. What is the difference between final, finally, and finalize?
**Tags:** `#Java-Core` `#Advanced` `#Tricky`  
**Difficulty:** ‚≠ê‚≠ê‚≠ê  
**Answer:**
- **final**: Keyword for constants, prevent inheritance/override
- **finally**: Block that always executes after try-catch
- **finalize()**: Method called by garbage collector before object destruction

---

#### 15. Can we change the value of a final variable?
**Tags:** `#Java-Core` `#Basics` `#Tricky`  
**Difficulty:** ‚≠ê  
**Answer:** No for primitives. For objects, reference can't change but object state can change.

### **Object-Oriented Programming (OOP)**

#### 16. What are the four pillars of OOP?
**Tags:** `#Java-OOP` `#Basics`  
**Difficulty:** ‚≠ê  
**Answer:** Abstraction, Polymorphism, Inheritance, Encapsulation (APIE)

---

#### 17. What is encapsulation? Give an example.
**Tags:** `#Java-OOP` `#Basics`  
**Difficulty:** ‚≠ê  
**Answer:** Wrapping data and methods in a single unit (class). Use private variables with public getters/setters.

---

#### 18. What are access modifiers in Java?
**Tags:** `#Java-OOP` `#Basics`  
**Difficulty:** ‚≠ê  
**Answer:**
- **private**: Same class only
- **default**: Same package
- **protected**: Same package + subclasses
- **public**: Everywhere

---

#### 19. What is inheritance? Types of inheritance?
**Tags:** `#Java-OOP` `#Basics`  
**Difficulty:** ‚≠ê  
**Answer:** Acquiring properties from parent class. Types: Single, Multilevel, Hierarchical. (Multiple not supported in Java)

---

#### 20. Why doesn't Java support multiple inheritance?
**Tags:** `#Java-OOP` `#Advanced` `#Tricky`  
**Difficulty:** ‚≠ê‚≠ê  
**Answer:** Diamond problem - ambiguity when two parents have same method. Solved using interfaces.

---

#### 21. What is polymorphism? Types?
**Tags:** `#Java-OOP` `#Basics`  
**Difficulty:** ‚≠ê  
**Answer:** One name, many forms. Types:
- **Compile-time**: Method overloading
- **Runtime**: Method overriding

---

#### 22. Difference between method overloading and overriding?
**Tags:** `#Java-OOP` `#Basics` `#Tricky`  
**Difficulty:** ‚≠ê‚≠ê  
**Answer:**
- **Overloading**: Same name, different parameters, same class, compile-time
- **Overriding**: Same signature, different classes (parent-child), runtime

---

#### 23. Can we overload main method?
**Tags:** `#Java-OOP` `#Tricky`  
**Difficulty:** ‚≠ê‚≠ê  
**Answer:** Yes, but JVM calls only `public static void main(String[] args)`

---

#### 24. Can we override static methods?
**Tags:** `#Java-OOP` `#Tricky`  
**Difficulty:** ‚≠ê‚≠ê  
**Answer:** No. Static methods belong to class, not object. It's method hiding, not overriding.

---

#### 25. Can we override private methods?
**Tags:** `#Java-OOP` `#Tricky`  
**Difficulty:** ‚≠ê‚≠ê  
**Answer:** No. Private methods are not visible to subclasses.

---

#### 26. What is abstraction?
**Tags:** `#Java-OOP` `#Basics`  
**Difficulty:** ‚≠ê  
**Answer:** Hiding implementation details, showing only functionality. Achieved using abstract classes and interfaces.

---

#### 27. Difference between abstract class and interface?
**Tags:** `#Java-OOP` `#Advanced` `#Tricky`  
**Difficulty:** ‚≠ê‚≠ê  
**Answer:**
- **Abstract Class**: Can have concrete methods, constructor, any variables, extends (single)
- **Interface**: Only abstract methods (Java 8+ allows default), no constructor, only constants, implements (multiple)

---

#### 28. Can abstract class have constructor?
**Tags:** `#Java-OOP` `#Tricky`  
**Difficulty:** ‚≠ê‚≠ê  
**Answer:** Yes. Called when subclass object is created.

---

#### 29. Can interface have concrete methods?
**Tags:** `#Java-OOP` `#Advanced` `#Tricky`  
**Difficulty:** ‚≠ê‚≠ê  
**Answer:** Yes, from Java 8: default and static methods.

---

#### 30. What is a functional interface?
**Tags:** `#Java-OOP` `#Advanced`  
**Difficulty:** ‚≠ê‚≠ê  
**Answer:** Interface with exactly one abstract method. Used for lambda expressions. Example: Runnable, Callable.

- **HashMap**: Not thread-safe

#### 39. What is the load factor in HashMap?
